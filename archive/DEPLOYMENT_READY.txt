STEP-BY-STEP PAYMENT FLOW:

1️⃣  Customer opens app → Sees "🟢 Toilet Free"
    └─ Clicks "💳 Pay to Enter"

2️⃣  Customer enters phone number → Clicks "Initiate Payment"
    └─ Frontend: POST /api/payments/create
    └─ Backend: Creates payment record (status="pending")

3️⃣  Customer confirms PIN on phone
    └─ PayPack: Payment confirmed

4️⃣  Frontend polls status every 2 seconds
    └─ GET /api/payments/status/{transaction_id}
    └─ Receives: "status": "successful", "command": "OPEN_DOOR"

5️⃣  Backend processes payment
    └─ Updates payment status = "completed" ✅
    └─ Adds revenue to toilet ✅
    └─ Marks toilet as occupied ✅
    └─ Logs payment event (triggers SSE) ✅

6️⃣  Frontend shows "✅ DOOR OPENING..."
    └─ Dashboard auto-refreshes via SSE

7️⃣  Dashboard updates
    └─ Total Revenue: +100 RWF ✅
    └─ Toilet Status: OCCUPIED ✅
    └─ Owner sees everything in real-time ✅

8️⃣  ESP32 polls for door command
    └─ GET /api/hardware/payment-check/1
    └─ Receives: "command": "OPEN_DOOR"

9️⃣  Door opens automatically
    └─ Servo: 0° → 90° (opens)
    └─ Waits: 10 seconds
    └─ Servo: 90° → 0° (closes)

🔟 Customer enters → Uses toilet → Exits
    └─ Button or sensor triggers exit
    └─ Door opens again: 0° → 90°
    └─ Door closes: 90° → 0°
    └─ Toilet marked as free

↻  System ready for next customer
    └─ Dashboard shows "🟢 Toilet Free"
    └─ Revenue saved ✅
